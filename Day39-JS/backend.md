# What is Backend?

## Definition
The backend refers to the server-side of a web application. It's the part of the application that users don't see directly in their browser.

## Key Responsibilities
- **Database Management**: Store and retrieve data
- **Business Logic**: Process requests and apply rules
- **Authentication & Authorization**: Handle user login and permissions
- **API Development**: Create endpoints for frontend to communicate with
- **Server Operations**: Handle requests, perform calculations, and send responses

## Common Technologies
- **Languages**: Node.js, Python, Java, PHP, C#
- **Frameworks**: Express.js, Django, Spring Boot, Laravel
- **Databases**: MySQL, MongoDB, PostgreSQL, Firebase
- **Servers**: Apache, Nginx

## Backend vs Frontend
| Aspect | Backend | Frontend |
|--------|---------|----------|
| **Visible** | No (server-side) | Yes (browser-side) |
| **Language** | Various (Node.js, Python, etc.) | JavaScript, HTML, CSS |
| **Role** | Process data & business logic | Display UI & handle user interaction |
| **User Access** | Not directly accessible | Directly accessible |

## Example Flow
1. Frontend sends request to backend
2. Backend processes the request
3. Backend queries database if needed
4. Backend sends response back to frontend
5. Frontend displays the data to user
