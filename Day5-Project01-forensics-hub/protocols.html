Phase 1 - Identification & Acquisition

Isolate the target system from the network immediately to prevent remote wiping or lateral movement. Acquire volatile data (RAM) first, followed by bit-for-bit physical imaging of non-volatile storage using hardware write-blockers. Calculate and record initial cryptographic hashes (SHA-256) for all acquired images to ensure zero tampering.

Phase 2 - Analysis & Extraction

Mount forensic images strictly as read-only. Recover unlinked (deleted) files, analyze file system artifacts (MFT, registry hives), and timeline system events. Correlate network logs with host-based artifacts to trace the intrusion vector.

Phase 3 - Documentation & Reporting

Maintain a strict chain of custody document detailing every transfer of evidence. Generate a final report detailing the scope of the breach, indicators of compromise (IOCs), the timeline of events, and actionable remediation steps. Ensure all analytical findings are objectively reproducible.
